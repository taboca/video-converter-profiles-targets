# Memo Style Guide

## File Naming Pattern

* `README_MEMO_YYYY_MM_DD_NNN_slug.md`
* `YYYY_MM_DD` is the date.
* `NNN` is a three-digit sequence (e.g., `100`, `110`).
* `slug` is a descriptive, lowercase, snake_case title.

## Markdown Structure

* H1 title in Title Case that mirrors the slug (single `#`).
* Two bullet lines immediately after the H1:
  * author name
  * date in `YYYY-MM-DD`
* Sections use `##` headings, with optional `###` subheadings.
* Content uses short paragraphs and bullet lists.
* Bullet lists are surrounded by blank lines (blank line before and after the list).
* Sections are separated by blank lines.
* A "To-Do Plan" section uses bullets, typically separated by blank lines.
* A "Changes" section lists concrete edits.
